How to Download and Run Minorthird
 
Before you can download minorthird, java and ant must be installed.  Java can be found at http://java.sun.com.  Make sure that you download and install the SDK.  
Ant can be downloaded at http://ant.apache.org .  
 
Once you have downloaded both java and ant: (Note: Directions on how to do the following are below under “Setting Environment Variables”)
1.           Define JAVA_HOME 
2.           Define ANT_HOME
3.           Add $JAVA_HOME/bin and $ANT_HOME/bin to your PATH
4.           Define MINORTHIRD variable (optional for windows)
 
Setting Environment Variables (including your PATH)
Windows:
1.             Follow: Start → Settings → Control Panel
2.             If you are using Windows XP, click on Switch to Classic View on the left
3.             Double click on System
4.             Click on the Advanced tab
5.             Click the Environment Variables button near the bottom
6.             The top box should be labeled user variables and there may or may not be a variable already labeled PATH, JAVA_HOME or ANT_HOME.  
a.                    If the variable you are looking for is already there, highlight it and click Edit; otherwise click New, and add the variable name and location.
7.             Add the complete path of your java bin and ant bin to your PATH separated by semicolons.  Your addition should look something like this: JAVA_HOME\bin;ANT_HOME\bin
 
Linux (Cygwin) with bash shell:
Type these commands:
1.             % export JAVA_HOME=PATH OF j2sdk1.4.2_04 directory
2.             % export ANT_HOME= PATH of apache-ant-1.6.1 directory
3.             % export PATH=$JAVA_HOME/bin:$ANT_HOME/bin:PATH
4.             % export MINORTHIRD= PATH of minorthird directory
 
 
Here are three ways you can download minorthird:
1)      Download jar file from http://sourceforge.net/projects/minorthird

- This option is the fastest and easiest to set up, only java is needed.  It is good for people who do not want to look at the source code and do not need to update often.

2)      Download source from http://sourceforge.net/projects/minorthird

- This option is harder to set up than the jar file, you will need java, ant, to change your path, and compile the code, but is easier than setting up CVS.  I would recommend this option for those who would like to see the source code, but want an easier setup, do not want to update often, and are not interested in submitting changes to minorthird.

3)      Checkout from CVS

- This is the hardest to set up, but has the most advantages.  You can see the source code, update frequently and easily, and check in changes you have made.  I recommend this option for minorthird developers, and those who already have CVS set up.

 

Downloading from CVS:

Note: If you would like to submit changes to minorthird email William Cohen, wcohen@cs.cmu.edu, and Cammie Williams, cammie@cmu.edu, to get a personal account on raff.

  1. Download CVS, we suggest using tortoise CVS, which can be obtained at http://www.tortoisecvs.org/.  (If you are using cygwin, make sure to add CVS to your path and that ssh is installed.  Note: ssh is not installed under the default packages in the cygwin setup, it is suggested that you install all cygwin packages.)
  2. Move to the directory where you would like minorthird
  3. Set these environment variables:
    1. CVS_RSH=ssh
    2. cvsroot=:pserver:anonymous@raff.cald.cs.cmu.edu:/usr1/cvsroot (or cvsroot=:pserver:USER_NAME@raff.cald.cs.cmu.edu:/usr1/cvsroot)
  4. Type these commands:
    1. cvs login
    2. cvs checkout minorthird
    3. cvs update –dP (the –dP is needed to remove deleted-from-cvs files and directories)
  5. Alternatively, you can type: (Note: this option cannot be done anonymously)
    1. Export CVS_RSH=ssh
    2. cvs –d :ext:USER_NAME@raff.cald.cs.cmu.edu:usr1/cvsroot checkout minorthird
  6. The minorthird source tree should appear in your current directory.

 

 

How to compile Minorthird source code

  1. cd to MINORTHIRD
  2. Run the setup script which sets you classpath:
    1. In Windows type: script\setup
    2. In Cygwin type: source script/setup.sh
    3. In Linux type: source script/setup.linux
  3. To compile the code type:
    1. %ant build-clean
  4. To compile the javadocs type:
    1. % ant javadoc
  5. To run the tests to make sure things are working type:
    1. % ant tests
 
How to run the Minorthird jar file
1.      java –Xmx500M –cp MINORTHIRD_JAR.jar … (continue from here just as you would with the source code)

for example: java -Xmx500M –cp MINORTHIRD_JAR.jar edu.cmu.minorthird.ui.TrainTestClassifier –gui

2.      If you would like the jar file to be able to see and include files in your current directory:

Java –Xmx500M –cp “MINORTHIRD_JAR.jar;.” …
