Classification Tutorial

 

Using the GUI:

1)      Once your data is loaded into minorthird, you can run a classification task.  To run this type of task using the gui you can either type:

a.       java –Xmx500M edu.cmu.minorthird.Minorthird and click “Expt” under Classify Text  

b.      java –Xmx500M edu.cmu.minorthird.ui.TrainTestClassifier -gui

2)      When the experiment window appears, choose “Edit” under Parameter modification to set your parameters.  If you are unsure of how to set any of the parameters or what they mean, click the Parameters button in the help section of Execution controls. 

a.       The first parameter to set is the baseParameter, which is where you can choose what data to load, for this example you should choose lingspam.  If your repository is properly set up, you should find the data that you loaded by scrolling through the data sets that appear under repository key pull down list.  If you do not find the data that you loaded under repository key, close the parameters property editor and click the Repository button in the Help section of Execution controls.  This will tell you how your repository is currently set up and give instructions for loading the data.

b.      If you would like to save your experiment, click “Edit” saveParameters and type what you would like your parameters saved as.  Saving is optional.

c.       The next parameter you must specify is the signal.  You must either specify either spanProp or spanType.  For this experiment you want to set spanType to either real or span.

d.      The splitter parameter is initially set to the default (Random Splitter), but can be changed.  To change the splitter, click the Edit button next to splitterParameters.  To choose which splitter you would like to use, choose one from the pull down list labeled splitter.  To change the properties of this splitter (ex: what percentage of data should be used for training and what percentage should be used for testing) click the Edit button next to splitter.  Check the show details checkbox if you would like to see the results from the test partitions as well as the overall evaluation after the experiment has finished running.

e.       The training parameters are also assigned default values, but these values can be changed by clicking the Edit button beside trainingParameters.  Once training parameters properties window appears, you can choose a feature extractor from the pull down list.  To change the properties of the feature extractor, click the Edit button beside feature extractor.  To change the learner, choose a learner from the pull down list and click Edit to change the properties of this learner.  The label of the predicted output can be put in the output textbox.  Check showData if you would like to see an additional window the data from the training set.

3)      Once all you parameters are saved, you can press Start Task under execution controls.  The task will vary in the amount of time it takes depending on the size of the data set and what learner and splittler you choose.  When the task is finished, the error rates will appear in the Error messages and output text area along with the total time it takes to run the experiment.

4)      Now that you have run the experiment, you can look at your results.  In order to look at the details of your results, click the View Results Button in the Execution Controls section.  Click on the Overall Evaluation tab at the top and the Evaluation tab below that to view your results.  The summary tab shows you the results that were printed in the output window when you ran the experiment (it shows you the numbers like error rate and F1.)  The Precision/Recall tab show you the graph of Recall vs. Precision for this experiment.  The Confusion Matrix tab shows you how many things the classifier predicted as positive that are positive and how many that it predicted as positive that are negative and visa versa.

5)      Press the Show Labels button if you would like to view the input data for the classification task.

6)      Press the “Clear Window” button to clear all output from the output and error messages window.

 

Using the Command Line:

1)    To get started using the command line for a classification experiment type:

                        java –Xmx500M edu.cmu.minorthird.ui.TrainTestClassifier –help

            This will list all the command line arguments that you can use.

Note: You can enter as many command line arguments as you like along with the –gui argument.  This way you can use the command line to specify the parameters that you would like and use the gui to set any additional parameters or view the results.

2)      The first thing you probably want to enter on the command line is the data you would like to train or train/test on.  To do this type –labels and the repository key of the dataset you would like to use.  For this experiment you should type:            –labels lingspam

3)      Now you either want to specify the dataset you would like to test on by typing –test repositoryKey or which splitter you would like to use (this is for when you want to train and test on the same data set.)  To specify the splitter, type –splitter SPLITTER.  Here you can type: -splitter “new RandomSplitter()”

4)      The next necessary parameter to name is either spanProp or spanType.  To specify this parameter, type –spanType TYPE.  For this dataset TYPE can either be real or spam, so type: -spanType real or –spanType spam.

5)      Other parameters you may want to specify are: the learner (-learner), the feature extractor (-fe), or whether to save.  Use the –help command for descriptions and examples of these parameters.  If you are unsure of what learners or feature extractors to use, use the –gui command so that you can see the list of learners and feature extractor available (under trainingParameters).  For this experiment, you can type: -learner “new NaiveBayes()” and                                                       –fe “new Recommended.DocumentFE()”
