Extraction Tutorial

 

For this example we will use enron01_names as the training data and enron2_names as the testing data.

 

Using the gui:

1)      Once your data is loaded into minorthird, you can run a classification task.  To run this type of task using the gui you can type:

java –Xmx500M edu.cmu.minorthird.ui.TrainTestExtractor –gui –labels enron01_names –test enron2_names

Note:  Because we are using a specific training and testing set, we must define these data sets on the command line because the gui is not capable of specifying two different data sets

2)      When the experiment window appears, choose “Edit” under Parameter modification to set your parameters.  If you are unsure of how to set any of the parameters or what they mean, click the Parameters button in the help section of Execution controls. 

a.       The first parameter to set is the baseParameter.  However, in this case the training and testing set were set on the command line, so you do not have to edit this parameter.  In other cases if you do want to set or modify this parameter, click on Edit so that the Property Editor for BaseParameters window pops up.  If your data is properly loaded, you should be able to find the label of your data on the pull down list next to repositoryKey.  If you do not find your data, your repository may not be set up correctly, and you should close the current window and the property editor window and press the Repository button in the help section.

b.      If you would like to save your experiment, click “Edit” saveParameters and type what you would like your parameters saved as.  Saving is optional.

c.       The next parameter you must specify is the signal.  You must either specify either spanProp or spanType.  For this experiment you want to set spanType to name.

d.      In this experiment, we have already designated training and testing data sets, so you do not need to specify a splitter.  In other cases, the splitter parameter is initially set to the default (Random Splitter), but can be changed.  To change the splitter, click the Edit button next to splitterParameters.  To choose which splitter you would like to use, choose one from the pull down list labeled splitter.  To change the properties of this splitter (ex: what percentage of data should be used for training and what percentage should be used for testing) click the Edit button next to splitter.  Check the showTestDetails checkbox if you would like to see the results from the test partitions and compare true_names with predicted values as well as the overall evaluation after the experiment has finished running.

e.       The training parameters are also assigned default values, but these values can be changed by clicking the Edit button beside trainingParameters.  Once training parameters properties window appears, you can list annotators to embed in the feature extractor (optional).  To change the learner, choose a learner from the pull down list and click Edit if you would like to change the sequenceClassifierLearner, spanFeatureExtractor, or taggingReduction of this learner.  These parameters all have defaults, so they do not have to be changed.  The name of the mixup code to be run before extracting features can be put in the textbox beside mixup (optional).  The label of the predicted output can be put in the output textbox. 

3)      Once all you parameters are saved, you can press Start Task under execution controls.  The task will vary in the amount of time it takes depending on the size of the data set and what learner and splittler you choose, but extraction tasks usually take a minute or two.  When the task is finished, the error rates will appear in the Error messages and output text area along with the total time it takes to run the experiment.

4)      Now that you have run the experiment, you can look at your results.  In order to look at the details of your results, click the View Results Button in the Execution Controls section.  Click on the Evaluation tab to see the precision rates of the experiment.  If you checked the showTestDetails box while editing the splitter parameter, there will be a Full Test Set tab.  When you click on this tab, you can compare whatever is labeled (in this case name) to what the learner predicted.  When comparing green mean true positive, blue means false negative, and yellow mean false positive.  You can also click on the spanTypes tab and select a color and a spam type to highlight.  Make sure that you reset controls before highlighting or comparing.  Once you have made you selection, click Apply to see your selection.  If you want to view the precision rates of your experiment, click the Evaluation tab at the top of the window.

5)      Press the Show Labels button if you would like to view the input data for the extraction task.

6)      Press the “Clear Window” button to clear all output from the output and error messages window.

 

Using the Command Line:

1)    To get started using the command line for a classification experiment type:

                        java –Xmx500M edu.cmu.minorthird.ui.TrainTestExtractor –help

            This will list all the command line arguments that you can use.

Note: You can enter as many command line arguments as you like along with the –gui argument.  This way you can use the command line to specify the parameters that you would like and use the gui to set any additional parameters or view the results.

2)      The first thing you probably want to enter on the command line is the data you would like to train or train/test on.  To do this type –labels and the repository key of the dataset you would like to use.  For this experiment you should type:            –labels enron01_names

3)      Now you either want to specify the dataset you would like to test on by typing –test repositoryKey or which splitter you would like to use (this is for when you want to train and test on the same data set.)  To specify the splitter, type –splitter SPLITTER.  Here we are using fixed training and testing data so you can type:    –test enron2_names

4)      The next necessary parameter to name is either spanProp or spanType.  To specify this parameter, type –spanType TYPE.  For this dataset TYPE can either be real or spam, so type: -spanType name.

5)      Other parameters you may want to specify are: the learner (-learner) or whether to save.  Use the –help command for descriptions and examples of these parameters.  If you are unsure of what learners, use the –gui command so that you can see the list of learners and feature extractor available (under trainingParameters).  For this experiment, you can type:

-learner “new VPHMMLearner(new CollinsPerceptronLearner(1,5), new Recommended.TokenFE(), new InsideOutsideRedution())”

As you can see from this example, the sequenceClassifierLearner, spanFeatureExtractor, and taggingReduction are defined with the learner.  If you would like to see the options for these variables, use the –gui command.  Once the parameter modification window pops up, click Edit under Parameter Modification and click Edit next to training Parameters.  To see what learners are available, scroll through the pull down list next to learner.  Once you have chosen a learner, click the Edit button next to learner to choose your sequenceClassifierLearner, spanFeatureExtractor, and taggingReduction.  To edit any of these training parameters, press the Edit button next to them.

6)      Optional parameters to define are –mixup, -embed, or –output.  Use the –help command to learn more about these parameters.  –output is set to the default _prediction, so you only need to set this parameter if you would like to the name of the property learned.
